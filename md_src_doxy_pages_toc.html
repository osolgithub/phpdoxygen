Here I elaborate the default table of contents of a doxygen documentation

Default contents are

Default contents of a basic doxygen documentation are

  • Project name
    • Main page marked with with **'mainpage'** tag. could be php or md
    • other markdown pages
      Pages in sub folders will come first
    • Pages
      ie other pages marked with'page' tag. could be php or md
      • Sections made with comination of tableofcontents & section tags
        • subsections made with comination of subsection tags below section tab above
  • Files
    • File List
      ie folder structure of files with documentation
      if an new extension eg: html or js is to be shown, add thenm to "FILE_PATTERNS" under "Experts" tab >> Inputs
      if source codes are to be shown, check "SOURCE_BROWSER", under "Experts" tab >> Source Browser
      PS markdown files will not be listed
    • Globals

More advanced documentation with OOP/Classes will contain

  • Namespaces
  • Data structures
